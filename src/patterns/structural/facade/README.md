# 外观（Facade）模式

- 定义/意图：为复杂子系统提供一个统一、简化的高层接口，隐藏对象创建、依赖管理与调用顺序等细节，客户端仅与门面交互。

- 适用场景：
  - 对接复杂库/框架，仅需其中少量能力时，用门面封装常用路径。
  - 子系统分层，需要为每层提供稳定入口、降低层与层之间耦合。
  - 以门面兜住遗留系统，对外保持稳定契约，内部可渐进重构。

- 优点：
  - 简化使用、降低学习成本。
  - 降低耦合，限制变化传播范围。
  - 便于在单入口处理日志、监控、权限、事务等横切关注点。

- 缺点：
  - 可能引入一层抽象（简单场景会显得过度设计）。
  - 门面容易膨胀，应控制边界并保持接口内聚；必要时允许旁路直连子系统。
  - 子系统频繁变动时，门面也需同步维护。

- 项目示例：`src/patterns/structural/facade/` 中 `ComputerFacade.boot()` 统一协调 `CPU.start()` / `Memory.load()` / `Disk.read()`；也可调用 `facadeDemo()` 查看效果。
